# Purchase Recommendation Rules

Flavor OS should recommend purchases based on flavor architecture, not hype.

## Criteria

- Missing sensory vectors
- Missing role types
- Versatility
- Compatibility with user's existing inventory
- Importance in LISSO mixology
- Domestic availability
- EC inventory status

## Example Logic

If the user has many sweet fruit flavors but lacks acid:

- Recommend Lime
- Recommend Lemon
- Recommend Grapefruit

If the user lacks structural/herbal/saline elements:

- Recommend Olive
- Recommend Nettle
- Recommend Green Tea
- Recommend White Tea

If the user lacks beginner-friendly dessert body:

- Recommend Vanilla
- Recommend Yogurt
- Recommend Cream
- Recommend Biscuit

## Output Requirements

Purchase recommendations should include:

- Reason
- What gap it fills
- Which existing flavors it improves
- Priority
- EC link if available
