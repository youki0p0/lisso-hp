# ShishaOS Overview

ShishaOS is not merely a recipe app.

It is a sensory sharing operating system for shisha mixology.

## Core Concept

```txt
ShishaOS
= inventory management
+ sensory vector database
+ LISSO mixology theory
+ recipe generation AI
+ substitution / reconstruction engine
+ EC purchase recommendation
+ X sharing
```

## Main User Experience

1. User registers their shisha flavor inventory.
2. ShishaOS recommends mixes from the user's inventory.
3. User saves and shares a recipe.
4. Another user opens the shared link.
5. ShishaOS reconstructs the recipe using that viewer's inventory.
6. Missing elements are suggested as EC purchases.

## Core Value

The strongest value of ShishaOS is the curated Flavor Master Database.

The database should be based on sensory evaluations by LISSO, converted into structured numerical data by AI, and verified by a human.

## Key Principle

AI must not invent taste from flavor names alone.

AI may convert human sensory descriptions into structured data, but human verification is required before public use.
